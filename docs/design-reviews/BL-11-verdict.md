# BL-11 design-review verdict — user's own beaver art, slimmed animation set

Date: 2026-07-14
Verdict: **PASS**

## Scope

Replace the app's beaver sprite art (previously programmatic pixel maps,
generated by an AI vision model from a reference image) with the user's own
generated images, and slim the animation set from `idle/walk/run/sleep/react`
down to `idle/walk` only. Adult stage art doesn't exist yet — falls back to
teen. Lodge (hatch) art is untouched.

## Ingest pipeline

`scripts/gen-sprites/ingest-images.mjs` (new, committed; plain `.mjs`, no new
npm deps). PNG decoding is hand-rolled: all 12 source files under
`assets-src/beaver/` (gitignored, copied in from `~/Downloads/`) are 8-bit
RGBA, non-interlaced (color type 6) — checked directly, not assumed — so the
decoder only supports that combination and throws on anything else.

Per-frame pipeline:
1. **Background removal**: flood-fill transparency from the four borders
   over pixels that are already near-transparent (alpha < 128), near-white
   (all channels ≥ 235), or near-black (all channels ≤ 25) — one predicate
   covers both "already-transparent PNG" and "opaque white/black backdrop"
   without per-file branching. Then a hard alpha threshold (< 128 → fully
   transparent, else fully opaque) over the *whole* frame, not just the
   flood-filled region — cleans up antialiased silhouette edges too ("feather
   nothing"). **Finding**: every one of the 12 source files already ships
   with a transparent background (all border/corner pixels alpha 0) — the
   flood-fill step is a no-op on this batch, but stays in the pipeline for
   any future opaque-background source images. Interior whites (teeth, eye
   shine) and blacks (eyes, nose) are never border-connected, so they
   survive untouched — verified visually in the capture below.
2. **Crop** to the opaque-pixel bounding box.
3. **Downscale**: premultiplied-alpha area-average box filter (exact
   fractional pixel-overlap weights) to a scale factor locked per stage —
   see "Scale factor" below.
4. **Place** on a 96×96 tile, bottom-aligned (feet on the tile's last row),
   horizontally centered.

### Scale factor (deviation from the literal spec, documented)

The spec's target content heights were baby ~72px, teen ~92px, "same scale
factor across all frames of a stage, locked from the tallest frame." Applying
that literally overflows the tile for baby: the walk-cycle source images
(`baby-to-right-{1,2}.png`) are a wide crouched/crawling pose — bbox
922×525px and 874×524px — vs. the idle pose's 537×587px. Locking scale to
idle's height alone (72/587 ≈ 0.1226) would scale the walk frame to ~113px
wide, overflowing the 96px tile and either clipping the tail or bleeding
into the neighboring sheet column.

**Auto-decision**: `computeStageScale` takes `min(targetHeight / maxFrameHeight,
tile / maxFrameWidth)` — the same "lock one scale factor per stage" contract
(no per-frame wobble), just computed from *both* dimensions across every
frame instead of height alone. Measured result:
- **baby**: width-constrained, scale = 0.1041 (idle content height ends up
  ~61px, not the literal ~72px target — the tradeoff for not clipping the
  walk pose).
- **teen**: height-constrained as the spec assumed, scale = 0.0949 (idle
  content height ~92px, on target; teen's widest frame is comfortably under
  96px at this scale, no overflow).

This is logged here per CLAUDE.md's "ambiguity about *how* → pick the
least-expansive interpretation" — least-expansive read as "don't ship
visibly broken/clipped art to hit an approximate (~) number."

## Deleted (programmatic pipeline superseded)

`scripts/gen-sprites/pixel-maps/{baby,teen,adult}.ts`,
`scripts/gen-sprites/import-codex.mjs`, `assets/sprites/beaver-adult.{png,json}`.
`build.ts`/`generator.test.ts` now only build/test the lodge sheet — the
beaver `SheetSpec` entries and their pixel-map imports are gone.
`sheet.ts`'s `BEAVER_ANIMATION_ORDER`/`BeaverAnimation` (bound to the old
idle/walk/run/sleep/react row order) are gone too — dead once nothing built
a beaver sheet that way. The tail-paddle guard test (checked the `t`/`T`
palette chars formed a flat silhouette in every frame) is deleted with it —
that guard only made sense against palette-authored pixel maps, and the
palette rule is explicitly waived for imported art. `ingest-images.test.ts`
covers the imported sheets instead: committed-sheet-matches-JSON-dimensions,
frames-non-empty, bottom-row-grounded, and re-run-is-byte-identical
(determinism) — the whole suite `describe.skipIf`s when `assets-src/` is
absent (fresh clone, gitignored source images).

## App changes

- **`sprites.ts`**: `loadSheet('adult')` falls back to `beaver-teen.png`
  (commented `// awaiting user art`) until real adult art exists.
- **`roam.ts`/`pet-config.ts`**: `run` and `sleep` phases/anims deleted
  outright (not just unused) — `Phase`/`AnimName` are now
  `idle | walk | climbUp | climbPause | climbDown` /
  `idle | walk`; `decideNext` only rolls climb-vs-walk (no sleep roll, no
  run-vs-walk split); `SLEEP_PAUSE_{MIN,MAX}_S`, `SLEEP_PROBABILITY`,
  `RUN_SPEED_PX_S`, `RUN_PROBABILITY` are gone from `pet-config.ts`, not
  zeroed.
- **`renderer.ts`**: `celebrate()` is a no-op (kept exported per the plan,
  for any future quip/evolution UI hook, but does nothing — no react row to
  play). Evolution sequence ends at `setStage(targetStage)`, no trailing
  `celebrate()` call. Hatch's `baby-appear` phase draws the `idle` row
  (was `react`).
- **Quips**: unaffected by construction — `detectors.ts`'s idle trigger
  reads `UsageTracker` token-delta zero-streaks (`idleSinceMs`), never
  `roam.ts` state. Confirmed no reference to roam/sprite state anywhere in
  `src/main/quips/`.
- **`PET_SCALE`**: 2 → 1 (`BEAVER_TILE_PX`: 48 → 96 — same 96px on-screen
  size as before, `48×2 = 96×1`). The lodge sheet kept its 48px native tile
  (untouched), so a new `LODGE_SCALE = 2` constant preserves its prior 96px
  on-screen size — `sprites.ts#drawFrame` now takes `scale` as an explicit
  per-call parameter (was a hardcoded `PET_SCALE` import) specifically so
  the pet draw path and the lodge/hatch draw path can each pass the right
  one. Every `hatchPosition()`/dirty-rect calc involving the lodge tile
  switched from `PET_SCALE` to `LODGE_SCALE`; the pet-sheet-drawing
  `baby-appear` branch and the normal roam draw kept `PET_SCALE`. Verified
  via the live hatch capture below (lodge and baby-appear both render at the
  same on-screen size, no jump at the handoff) and `roam.test.ts`'s bounds
  tests (which derive the clamp from `BEAVER_TILE_PX * PET_SCALE`
  dynamically, not a hardcoded 96, so they'd have caught a mismatch).
- **`SPRITE_FPS`**: 10 → 8, matching the ingested sheets' `fps` hint. The
  lodge sheet's own `lodge.json` still says `fps: 10` (untouched asset) —
  `SPRITE_FPS` drives both sheets' frame-advance cadence in the renderer
  regardless of what each JSON records (it always did; the JSON `fps` field
  turns out to be documentation only, read nowhere except the now-deleted
  `celebrate()`'s duration calc). Both 8 and 10 are inside CLAUDE.md's
  8–12fps sprite-cadence budget, so this is cosmetic.

`hatch.test.ts` needed no changes — neither references sprite sheets, scale,

## Deviations from the plan (with rationale)

1. **Scale-factor formula** (see "Scale factor" section above) — width
   floor added alongside the height lock to avoid clipping baby's wide walk
   pose; baby's actual idle content height (~61px) undershoots the literal
   ~72px target as a result.
2. **`ingest-images.d.mts`** (new, hand-written ambient type declarations)
   — `ingest-images.mjs` is plain JS (no TS syntax allowed per its own
   header comment: it must run directly under `node`, and Node's type
   stripping only applies to `.ts`/`.mts` files, not `.mjs`), but
   `ingest-images.test.ts` needs real types rather than implicit `any`
   under this repo's `noImplicitAny` strict tsconfig. A `.d.mts` companion
   (not `.d.ts` — TS's `bundler` module resolution maps `.mjs` imports to
   `.d.mts` declarations specifically) gives the test file types without
   adding any runtime code or a build step to the `.mjs` script itself.
   `scripts/gen-sprites/tsconfig.json`'s `include` glob extended to pick it
   up (`**/*.d.mts` alongside the existing `**/*.ts`).
3. **`npm run assets:ingest`** script added to `package.json`, parallel to
   the existing `assets:build` — not explicitly requested, but a one-line
   discoverability parity with the lodge pipeline's own script felt too
   cheap to skip.
4. **`docs/design-reviews/BL-3-contact-{baby,teen,adult}.png`** (stale
   contact sheets from the deleted programmatic pipeline) left in place —
   out of this item's scope; `build.ts` no longer regenerates them since it
   no longer builds beaver sheets at all, but deleting historical
   design-review artifacts wasn't asked for.
5. **CPU sample wasn't idle-phase-gated** (BL-4/BL-10 filtered samples to
   only count after ≥3s of continuous idle/sleep roam phase). BL-11 has no
   `sleep` phase anymore, and gating on `idle` alone over only a 10-13s
   window left too few qualifying samples to be meaningful — reported the
   raw mixed-phase average instead, labeled as such.
